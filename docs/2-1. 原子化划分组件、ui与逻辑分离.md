1. 什么是 metadata.json 及其作用？

metadata.json 是该应用项目的配置文件。它的主要作用包括：
- 应用标识：定义应用的名称 (name) 和描述 (description)，这些信息通常用于在平台展示或作为网页的元标签。
- 权限管理：这是最重要的功能。如果应用需要调用硬件接口（如摄像头 camera、麦克风 microphone）或获取地理位置 (geolocation)，必须
在此文件的 requestFramePermissions 数组中进行声明。

2. 文本溢出处理：使用已在 index.html 定义的 line-clamp-1 类，确保描述文本只显示一行，超出部分自动以 ... 结尾

4. ui与逻辑分离

问题：事件函数、纯函数、工具函数、自定义hooks区别？

解答：

- 纯函数/工具函数：输入 → 输出，永远相同，无任何副作用，即无状态的数学/逻辑运算；比如数据格式化（金额、日期）、正则校验、数学计算、复杂对象
处理（深克隆、归并）等；
- 事件函数：处理单次用户交互，习惯handleXxonXx命名，有状态变化，但通常无自己状态（即事件函数中无useState）；典型特征比如直接 
setXxx / fetch / console.log等；
- 自定义Hooks：复用状态逻辑，必须use开头，有状态变化；函数组件或自定义Hooks最顶层，不能在if/循环/嵌套函数里调用；典型特征比如
包含 useState / useEffect / useMemo 等等
- 总结：纯函数/工具函数与事件函数、自定义hooks区别是是否有状态变化、副作用；事件函数和自定义hooks区别是是否复用、是否遵循hooks调用规则（顶层
调用、不能在if/循环/嵌套函数里调用）、是否能调用其他hooks等；

12. 白屏问题自底向上常见原因分析？如何排查？为什么这么排查？

    白屏的本质：**js代码在渲染阶段前的某个阶段，因为某种致命错误强制中断/挂起了执行流程**。

    导致白屏的一些常见类型及示例：路径错误、依赖冲突、空模块/导出null等。
    - 路径错误：如index.html中入口文件（main.tsx）引用路径错误；采用默认导入（路径指向文件夹），而该文件夹下文件命名非vite默认的index文件命名等。
        - 导入路径指向的是文件夹时（比如import ... from './components/Modal'），构建工具vite会默认寻找该文件夹下的index.xx文件，找不到则报错。
        - 而导入路径具体指向了文件名时（比如import AdCard from './components/AdCard/AdCard.tsx'），那么不用命名为index.xx文件，vite不会报错
    - 依赖冲突：如外部映射将将react指向19版本，而本地依赖安装的是18版本，react依赖冲突导致报错，出现白屏。
        - React 的 Hook（如 useState）要求必须在同一个 React 实例的上下文中运行。
        - 普通的工具库（如 lodash）冲突，通常不会白屏；但是诸如框架核心库（React, Vue）、样式引擎（Styled-components, Emotion）、路由库（React-Router多个 Router 实例冲突）、状态管理（Redux, MobX，版本不匹配可能导致状态分发失效）等具有破坏性变更或单例模式的库，版本冲突极高概率触发白屏。
    - 空模块/导出null：如A模块强依赖B模块，而B模块文件为空/导出为null/没有导出，导致一直处于链接阶段，出现白屏。
        - 强依赖：指 A 模块在顶层（Top-level）通过 import 直接引入了 B 模块。
